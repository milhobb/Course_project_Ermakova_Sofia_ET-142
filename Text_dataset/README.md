Кейс №57: Классификация спама по тексту электронного письма
Исходные данные: тексты электронных писем, преобразованные в вектор признаков на основе словаря из 5 000 слов. Разметка: спам / не спам.
Требуется: построить модель классификации, определяющую, является ли письмо спамом

Датасет: Spambase (https://archive.ics.uci.edu/dataset/94/spambase)

Авторы/владельцы: Mark Hopkins, Erik Reeber, George Forman, Jaap Suermondt (Hewlett-Packard Labs, 1999)

Тип задачи: бинарная классификация текстов.
Входные данные: 57 признаков для каждого письма
Выходные данные: бинарная метка класса (“0” означает не спам, “1” – спам).


Исходные тексты писем в датасете отсутствуют — данные представлены в виде 57 числовых признаков (частоты слов и символов, статистики заглавных букв). В связи с этим точные значения средней длины текста в символах и словах недоступны.
